File Structure

A common approach to the structuring of pytorch models is to split the overall model into three independent files.

The first objective is always to define the type of data the model will take as input, and transform it into pytorch tensors. This first task of data handling is handled in the 'data.py' file. The next general task is then to define the network's architecture. This is done in the file 'model.py'. Finally, the model must be trained, which is generally called as a script from the command line and defined in the file 'main.py'

These three files 'data.py', 'model.py', and 'main.py' can usually be defined independent enough that changes in one requires no change to the other. For example, if the processing of input data in 'data.py' were to be changed slightly, the model defined in 'model.py' should require no change.
